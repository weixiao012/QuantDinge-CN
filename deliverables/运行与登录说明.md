# QuantDinger 中文版运行与登录说明

## 本机入口

- Web：`http://localhost:8888/#/user/login`
- Mobile H5：`http://localhost:8889`
- 后端健康检查：`http://localhost:8888/api/health`

## 管理员账号

默认用户名：`quantdinger`

密码属于本机部署密钥，不写入公开仓库。公网部署时应在服务器环境变量或密钥管理器中重新生成并保存。

## 当前运行状态

当前完整平台由 Docker 容器运行：

- `quantdinger-frontend`：中文前端镜像 `quantdinger-frontend-cn:local`
- `quantdinger-backend`：QuantDinger 后端
- `quantdinger-db`：PostgreSQL
- `quantdinger-redis`：Redis
- `quantdinger-mobile`：移动端 H5

## QQ 邮箱验证码登录

QQ SMTP 基础参数已准备好，仍需 QQ 邮箱地址和 SMTP 授权码：

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\22212\Documents\QuantDinge\project001-QuantDinger中文版\scripts\configure-qq-smtp.ps1
```

授权码不是 QQ 密码，需要在 QQ 邮箱后台开启 SMTP 服务后生成。

当前中文版默认进入“验证码登录”。如果 QQ SMTP 尚未配置，页面会显示明确提示并禁用“发送验证码”按钮，避免用户误以为验证码系统坏了。
