# 公网永久部署方案

本项目不是纯静态网页，包含前端、后端、Postgres 和 Redis。要做到“所有人都可以一直使用”，必须满足两个条件：

1. 承载机器长期在线：推荐云服务器/VPS。
2. 有公网入口：推荐域名 + Cloudflare 命名隧道，或服务器公网 IP + 反向代理。

## 方案 A：云服务器/VPS

适合真正长期使用。把仓库拉到服务器后运行 Docker Compose，让服务器的 80/443 对外开放。

基本步骤：

```powershell
git clone <你的GitHub仓库地址>
cd QuantDinge-CN\project001-QuantDinger中文版\source\QuantDinger
docker compose -f docker-compose.ghcr.yml -f docker-compose.cn.yml up -d
```

然后把域名 DNS 指向服务器公网 IP，并用 Nginx/Caddy 做 HTTPS 反向代理到 `127.0.0.1:8888`。

## 方案 B：Cloudflare 命名隧道

适合没有公网 IP，但有 Cloudflare 账号和域名的情况。注意：如果用本机承载，本机必须一直开机，Docker 容器也必须一直运行。

先登录 Cloudflare：

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\22212\Documents\QuantDinge\project001-QuantDinger中文版\scripts\cloudflare-login.ps1
```

再创建命名隧道并绑定域名：

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\22212\Documents\QuantDinge\project001-QuantDinger中文版\scripts\create-cloudflare-tunnel.ps1 -TunnelName quantdinger-cn -Hostname quantdinger.example.com
```

把 `quantdinger.example.com` 换成你自己的域名。

## 不能作为永久方案

`cloudflared tunnel --url http://localhost:8888` 会生成临时地址，但地址会变，进程一停就不可用，不满足“所有人一直使用”。
