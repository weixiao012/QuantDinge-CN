# 全局复利与踩坑日志

## 用途

记录犯过的错误、踩过的坑、走过的弯路，以及未来可以持续复用的方法。这里写跨项目经验；单个项目的详细日志放在该项目目录的 `logs/` 中。

## 记录格式

```text
日期：
项目：
问题：
原因：
解决方法：
可复用结论：
```

## 已知规则

- 如果用户给出 GitHub 仓库名但克隆失败，应先核对拼写、大小写和实际可访问仓库名。
- Windows PowerShell 环境中，如果本机规则要求使用工具前缀，应先确认该工具是否在 PATH；不在 PATH 时使用绝对路径。
- 新项目启动前必须先检查 `AGENTS.md`、`全局工作台.md`、`全局复利与踩坑日志.md`、`新项目创建SOP.md`。
